# BITESHA Launch Strategy

> Ecosystem-first. Infrastructure before token. Users before liquidity.

---

## Core Principle

```
You do NOT launch a token to get users.
You launch users to justify a token.
```

---

## Launch Phases

### Phase 0 — Foundation ✅

- [x] Brand: BITESHA + BTSH
- [x] Tokenomics defined
- [x] EVM L2 network selected (Base / Arbitrum)
- [x] System architecture designed
- [x] Decision framework (RFC-0001) established
- [x] Smart contract skeleton complete

### Phase 1 — Build

- [ ] ERC-20 BTSH contract
- [ ] Vesting contracts
- [ ] Treasury multisig
- [ ] Governance contracts
- [ ] Testnet deployment
- [ ] Test coverage ≥ 90%

### Phase 2 — Private Ecosystem Beta

- [ ] BITESHA Wallet (beta)
- [ ] Backend API
- [ ] Internal beta users (builders only)

> ❌ No public token  ❌ No hype  ✅ Real product, real usage

### Phase 3 — TGE (Token Generation Event)

- [ ] Mint 1,000,000,000 BTSH (genesis, one-time)
- [ ] Allocate by tokenomics
- [ ] Activate vesting contracts
- [ ] Activate treasury

> Token is issued ONLY after infrastructure is ready.

### Phase 4 — DEX Listing

**Target DEXes:**
- Uniswap v3 on Base
- Aerodrome on Base
- SushiSwap (backup)

**Liquidity model:**
- Portion of treasury → LP (locked minimum 12 months)
- No artificial wash trading

### Phase 5 — CEX Listing (Optional)

Approach: create demand → exchanges reach out.

**Tier 2 first:** Gate.io, KuCoin, MEXC  
**Tier 1 later:** Binance, Coinbase (when ecosystem traction is real)

We do not buy listings.

### Phase 6 — Ecosystem Expansion

- BITESHA Wallet v1
- BITESHA ID (identity layer)
- Developer SDK public release
- API platform
- Governance DAO activation
- Ecosystem app marketplace

---

## Liquidity Strategy

### Prohibited
- Pump and dump
- Fake volume / wash trading
- Early market manipulation

### Permitted
- Controlled LP provision
- Long-term liquidity locking
- Gradual expansion tied to ecosystem growth

---

## Go-To-Market Sequence

```
Phase A: Builders first
  → developers, crypto natives, early contributors

Phase B: Users
  → wallet users, ecosystem apps, utility adoption

Phase C: Market
  → exchanges, liquidity, broader crypto audience
```
